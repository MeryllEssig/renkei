## Rules 

* Always code in TDD
* Always run tests before giving 
* Always do a conventional commit after every atomic step
* When you implement a task from a plan file in `./plans`, you must tick it to keep trace of progress.
* Everything written in this repo should be in english.
* Always use [PRD](./PRD.md) as reference file.
* If you make a decision that changes the plan's execution (e.g. deferring a task to a later phase), update the plan accordingly. The plan must always stay aligned with the implementation goals. Report any such decisions to the user at the very end.
* If you modify the PRD, always place the content in the appropriate location:
    * In the index PRD.md
    * In an existing thematic sub-file in ./doc/prd/
    * In a new thematic sub-file in ./doc/prd/, updating the index accordingly.
* Maintain a good folder structure to improve retrievability.
